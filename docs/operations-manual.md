# Oussouri Caviar HUB — 平台运营手册（Operations Manual）

> 更新：2026-07-05　适用版本：main @ P2 批次 1
> 读者：平台管理员、居间专员（Broker）、客服、财务等内部运营人员

## 1. 角色与权限一览

| 角色 | 登录后看到 | 核心职责 |
|---|---|---|
| ADMIN 管理员 | 管理后台 `/admin` | 入驻审核、产品审核、订单监控 |
| SUPER_ADMIN 超管 | 同上 + 穿透审批 | 权限分配、审批高敏穿透、接收全部穿透抄送 |
| BROKER 居间专员 | 居间作业台 `/broker` | 商机跟进、撮合、居间代下单 |
| FINANCE 财务 | 账本接口 | 对账、退款、佣金 |
| SUPPLIER / BUYER | 各自工作台 | 外部用户，注册后需审核 |

**铁律**：所有运营动作都会留审计（操作人/时间/IP/前后值）。查看任何客户真实身份必须走穿透申请，事由会抄送超管——这是保护你也是保护平台。

## 2. 日常运营流程

### 2.1 入驻审核（每日）

管理后台 → 入驻审核：队列显示申请方的**平台代码、类型、国家、提交时间**（默认脱敏）。审核要点：
- 供应商：营业执照/SC 证/CITES 资质齐全且在有效期（P2 后证书到期自动告警）；
- 采购商：VAT 税号/进口资质可核验；
- 通过 → 对方即可登录交易；拒绝 → 组织转 INACTIVE，可备注原因。
需要核对真实公司名时：点穿透申请，填写事由（如"核验营业执照一致性"），低敏字段即时放行 30 分钟窗口。

### 2.2 产品上架审核

供应商提交后进入 PENDING_REVIEW。检查：品类/品种/等级与描述一致、HS 编码正确、无联系方式夹带、（P2 后）溯源批次与证书关联完整。通过即上架（ACTIVE），拒绝退回草稿。

### 2.3 订单与履约监控

订单状态机：`PLACED → PAID_ESCROW → CONFIRMED → PREPARING → SHIPPED → (IN_CUSTOMS → CUSTOMS_CLEARED) → DELIVERED → COMPLETED`。运营关注点：
- PLACED 超 24h 未支付：库存预留自动过期释放，无需干预；
- PAID_ESCROW 长期未接单：联系供应商（经平台通道）；
- DELIVERED 后 48h 争议期届满自动分账（供应商到账 = 总额 − 8% 佣金）；
- 争议（DISPUTED）：资金自动冻结，客服调查后由管理员裁决（全额/部分退款/驳回），裁决驱动退款。

### 2.4 居间专员（Broker）作业 SOP

1. **看商机流**：作业台按紧迫度排序，四维分含义——匹配（供需契合度）、机会（预估成交额）、紧迫（临期/断货/截止日）、利润（佣金空间）；每条商机的 explanation 写明触发原因（如"31 天未下单 + 偏好品种有货"）。
2. **认领**：先认领再跟进，防撞单；
3. **联系买家**：只能通过平台通道（站内信/平台代发；P2 后接 Twilio 一键外呼）。**绝不通过私人手机/微信联系客户**——系统层面你也拿不到对方号码；
4. **促成**：口头议定价格后点"居间代下单"，录入买家代码、SKU、数量、议定单价（系统拒绝低于阶梯价 70% 的报价）→ 系统自动锁货 24 小时并以平台名义向买家推送支付通知；
5. **闭环**：买家支付后订单进入正常履约流；商机自动关联订单；未成的商机标 LOST 并选原因（喂给未来的撮合模型）。

### 2.5 RFQ 反向招标运营

买家发布 → 全体供应商可见（买家仅显示代码）→ 供应商报价 → 买家接受自动成单。运营职责：临期未有报价的 RFQ 推给匹配供应商（撮合规则每 2 小时自动扫，也可在作业台手动"运行撮合"）。

### 2.6 财务（每日/每月）

- 日：`GET /finance/ledger` 核对账本与 Stripe Dashboard 余额（六科目：买家在途/托管/佣金/供应商应收/退款/手续费）；线下电汇订单人工核销水单；
- 月：佣金收入汇总（PLATFORM_COMMISSION 贷方合计）；退款复核。

## 3. 敏感信息与合规红线

1. 任何页面截图/导出发给外部前，确认只含平台代码；
2. 穿透查看用后即走，勿抄录到个人设备（审计可回溯到字段级）；
3. IM 中用户互发联系方式会被自动拦截（30 天内 3 次冻结其会话并告警）——冻结解除需管理员评估；
4. GDPR 删除请求：走匿名化流程（保留交易凭证骨架），联系开发执行；
5. 单证原件（CITES/健康证等）永不外发，只发脱敏水印副本（P2 功能上线后）。

## 4. 常见问题处置

| 现象 | 处置 |
|---|---|
| 用户反馈"消息发不出去" | 大概率触发联系方式拦截，指引其通过平台沟通；查 `/admin/message-blocks` 确认 |
| 买家付了钱订单没动 | 查 Stripe Dashboard 事件是否送达 webhook；对账任务会兜底，也可让开发重放事件 |
| 供应商说"库存明明有却下不了单" | 可售 = 在库 − 预留；查是否有未支付订单占用预留（24h 自动释放） |
| 商机流为空 | 正常：规则需要"流失买家/开放 RFQ × 匹配库存"同时存在；可手动运行撮合验证 |
| 误审核通过 | 管理员将组织改 SUSPENDED（保留数据），勿物理删除 |

## 5. 运营指标（建议每周看）

注册→审核通过率与时长｜上架 SKU 数｜GMV 与订单漏斗（PLACED→PAID 转化）｜商机转化率（WON/总数，按 Broker）｜RFQ 报价率与成交率｜IM 拦截次数（异常升高 = 绕单风险信号）｜争议率。P3 的 Analytics 看板上线前，用 DB 查询/账本导出人工统计。

## A. 注册与入驻审核（8 条）

| # | 步骤 | 预期 |
|---|---|---|
| A1 | 打开 `https://域名`，无登录浏览首页 | 深蓝门户加载正常：行情表、原产地、买家需求（仅 BY- 代码）、RFQ 大厅、五项数据带 |
| A2 | 注册 SUP-1：类型=供应商，公司名填真实全称，国家 CN，密码 ≥10 位 | 成功页显示平台代码 `SP-xxxxxx`，提示等待审核 |
| A3 | 注册 BUY-1：类型=采购商（进口商），国家 FR | 显示 `BY-xxxxxx` |
| A4 | 用 SUP-1 立即登录并尝试建产品 | 登录成功但创建产品被拒（组织未审核 PERM 提示）※当前实现登录可进工作台，下单/上架会被拦 |
| A5 | ADMIN 登录 → 管理后台 → 入驻审核 | 队列显示两条：仅代码/类型/国家，**不显示公司名** |
| A6 | 逐条点"通过" | 状态变 ACTIVE，队列清空 |
| A7 | 注册 BUY-2 后 ADMIN 点"拒绝" | 该组织不可交易（登录后下单报组织未激活） |
| A8 | 密码 <10 位注册 | 表单/接口报校验错误 |

（A7 后重新注册一个 BUY-2 并通过，供后续使用。）

## B. 身份防火墙（核心，9 条）

| # | 步骤 | 预期 |
|---|---|---|
| B1 | 匿名打开 市场 页 | 产品卡只显示 `SP-xxxxxx · 已验证`；价格区显示"登录查看批发价" |
| B2 | 匿名打开任一产品详情 | 无价格表；无任何公司名/联系方式 |
| B3 | BUY-1 登录后看同一详情 | 阶梯价表出现；供应商仍仅代码 |
| B4 | 浏览器查看页面源码/网络响应（详情与订单接口） | 搜索供应商公司全称 → **零命中** |
| B5 | BUY-1 与平台建 SUPPORT 会话，发送含手机号消息（如 `+33 7 49 88 49 70`） | 发送被拒，提示"请勿交换联系方式（第 1/3 次警告）" |
| B6 | 再发含邮箱、再发含微信号 | 均被拦；第 3 次后会话冻结，再发任何消息被拒 |
| B7 | 发送正常业务消息（另开会话）："订单 ORD-xxx 什么时候发货？" | **正常发出**（平台编码不误拦） |
| B8 | ADMIN 对 BUY-1 发起穿透申请（字段 companyName，事由 ≥10 字） | 低敏即时放行，返回 30 分钟窗口；凭 escalationId 读取到真实公司名 |
| B9 | ADMIN 申请高敏字段（taxId） | 状态 PENDING，需超管审批后才可读取；超管账号收到抄送通知 |

## C. 商品 / 溯源 / 库存（10 条）

| # | 步骤 | 预期 |
|---|---|---|
| C1 | SUP-1 → 供应商工作台 → 新建产品（品类 CAVIAR、HS 1604310000） | 列表出现 `PRD-xxxxxx`，状态 DRAFT |
| C2 | 为该产品添加 SKU（50g、阶梯价 0–50kg €320 / 50kg+ €302） | SKU 创建成功 |
| C3 | 点"提交审核" | 状态 PENDING_REVIEW；此时市场页**看不到**该产品 |
| C4 | ADMIN → 管理后台 → 输入 PRD 代码审核通过 | 状态 ACTIVE，市场页可见 |
| C5 | 溯源中心：登记基地（名称含"黑金河"）→ 建加工批次（批次号 PROC-T1，原料 500/成品 50） | 两者创建成功并显示在列表 |
| C6 | 用 CUSTOMS/ADMIN 的 token 对加工批次执行 QC：`POST /api/v1/admin/processing-batches/<id>/qc {"qcStatus":"QC_PASS"}` | 返回 QC_PASS |
| C7 | 库存入库：SKU + 批次号 LOT-T1 + 数量 100 + **加工批次号 PROC-T1** | 在库 100，预留 0 |
| C8 | （API）另建加工批次并 QC_FAIL，再用其入库 | 被拒：质检不合格不能入库 |
| C9 | 匿名访问 `https://域名/api/v1/products/<PRD代码>/trace` | 返回品种/工序/得率/效期；**响应中无"黑金河"等基地信息** |
| C10 | （API）SUP-1 给原料批次加用药记录但不填休药期 | 409 校验失败；填了未来休药期后创建加工批次 → 409 STATE_GUARD_FAILED |

## D. 下单与托管支付（8 条）

| # | 步骤 | 预期 |
|---|---|---|
| D1 | BUY-1 在产品详情输入数量 50 加购 → 采购台提交订单 | 生成 `ORD-YYYYMMDD-xxxxx`；金额 = **50 × €302 = €15,100**（验证 50kg 边界命中高档） |
| D2 | SUP-1 查看库存 | 在库 100 / 预留 50 |
| D3 | BUY-1 取消该订单 | 状态 CANCELLED；供应商库存预留恢复 0 |
| D4 | 重新下单 50kg → 点"支付（托管）" | 提示成功；订单变 PAID_ESCROW（测试环境为模拟支付） |
| D5 | SUP-1 工作台看到该订单 | 对手方仅显示 `BY-xxxxxx`；可见佣金字段 |
| D6 | SUP-1 点"接单" | 状态 CONFIRMED，出现"发货准备"面板 |
| D7 | BUY-2 登录尝试查看 BUY-1 的订单（改 URL/接口） | 看不到（403/空列表） |
| D8 | 下单数量低于 MOQ（如 0.5） | 被拒并提示 MOQ |

## E. 履约：单证守卫 / 冷链 / 清关（9 条）

| # | 步骤 | 预期 |
|---|---|---|
| E1 | D6 后 SUP-1 直接点"发货" | **拒绝**：需先登记运单 |
| E2 | 发货准备面板登记运单（承运商/运单号/HRB→CDG） | 提示运单已登记 |
| E3 | 逐件登记单证，先登记 6 件（留 HEALTH_CERT 不登） | 齐备度显示 6 绿 1 红，"缺件 1" |
| E4 | 此时点"发货" | 拒绝且提示缺 HEALTH_CERT |
| E5 | 补登 HEALTH_CERT → 发货 | 成功 SHIPPED；库存在库 50/预留 0 |
| E6 | （API）SUP-1 上传温度日志 3 条，其中 1 条 4.2℃：`POST /api/v1/logistics/orders/<ORD>/temperature-logs` | 返回 breaches:1；**BUY-1 通知里出现 TEMP_BREACH** |
| E7 | （API，CUSTOMS）创建出口报关单并迁移 SUBMITTED | 订单自动变 IN_CUSTOMS（买家订单页可见） |
| E8 | （API，CUSTOMS）报关单迁移 CLEARED | 订单自动变 CUSTOMS_CLEARED |
| E9 | BUY-1 点"确认签收" → 等待 ~10 秒刷新 | 订单自动 COMPLETED；ADMIN 用 `GET /api/v1/finance/ledger` 验证：佣金 €1,208 + 供应商应收 €13,892（=15,100 的 92%） |

## F. 居间撮合与代下单（8 条）

| # | 步骤 | 预期 |
|---|---|---|
| F1 | BUY-2 发布 RFQ（CAVIAR/DAU/60kg，两周后截止）后，BROKER 登录 → 作业台 → 点"运行撮合" | 商机流出现 RFQ_MATCH 商机（BUY-2 ↔ SUP-1），带四维评分与信号说明 |
| F2 | 检查商机卡 | 双方均为平台代码；无公司名 |
| F3 | 点"认领" | 卡片出现"我认领的"；另一 Broker 账号（可用 ADMIN）认领同一条被拒 |
| F4 | BUY-1 先在采购台"资料"里… ※当前联系人登记为 API：`POST /api/v1/party/contacts {"name":"主厨","phone":"+33 7 49 88 49 70","isPrimary":true}`（用 BUY-2 token） | 创建成功 |
| F5 | BROKER 对该商机点"一键外呼" | 提示"外呼已发起（号码对您不可见）"；通话记录区出现 DIALING 条目，**页面任何处无电话号码** |
| F6 | 点"居间代下单"：数量 20、单价 100 | **拒绝**（低于阶梯价 70% 底线） |
| F7 | 单价改 295 提交 | 提示意向单已生成；BUY-2 通知出现 ORDER_PAYMENT_LINK（24h 锁货）；SUP-1 库存预留 +20 |
| F8 | BUY-2 在采购台对该订单点"支付" | PAID_ESCROW；后续可走 E 组流程闭环 |

## G. RFQ 闭环（5 条）

| # | 步骤 | 预期 |
|---|---|---|
| G1 | SUP-1 工作台 → 可报价 RFQ | 看到 BUY-2 的 RFQ，买家仅显示 `BY-xxxxxx (FR)` |
| G2 | 行内输入 285 报价 | 显示"已报价"；重复报价按钮消失 |

| G4 | 点"接受并生成订单" | 生成 RFQ 订单 60×285=€17,100；RFQ 状态 ACCEPTED |

## H. 单证脱敏发送（4 条）

|---|---|---|
